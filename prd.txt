Managing  Cold  Starts  and  Inactivity  
The  primary  challenge  of  a  zero-cost  backend  is  the  tendency  for  services  to  sleep  after  15

This

"bad

must
 
wait
 
several
 
seconds
 
for
 
the

to
 
wake
 
up.
 Solution:  The  application  should  implement  a  "System  Waking"  UI  state.  When  the  client  app  
first
 
loads
 
on
 
Vercel,
 
it
 
should
 
immediately
 
send
 
a
 
GET
 
/health
 
request
 
to
 
the
 
backend.
 
While
 
the
 
VM
 
is
 
initializing,
 
the
 
React
 
app
 
displays
 
an
 
informative
 
message
 
explaining
 
that
 
the
 
"AI
 
engine
 
is
 
warming
 
up."
 
This
 
transparency
 
increases
 
user
 
retention
 
during
 
the
 
1-5
 
second
 
initialization
 
phase.

Gemini  1.5  Flash  tokens  for  native  PDF  processing  are  calculated  based  on  page  count  and  
resolution.
 
Passing
 
excessively
 
large
 
or
 
blurry
 
PDFs
 
can
 
lead
 
to
 
high
 
token
 
consumption
 
and
 
increased
 
latency.
 Solution:  The  frontend  should  perform  client-side  file  compression  and  check  that  the  PDF  
does
 
not
 
exceed
 
the
 
50MB
 
limit
 
before
 
attempting
 
an
 
upload.
 
Furthermore,
 
the
 
system
 
instructions
 
should
 
explicitly
 
tell
 
the
 
model
 
to
 
ignore
 
non-essential
 
visual
 
elements
 
like
 
profile
 
headers
 
or
 
background
 
graphics
 
to
 
focus
 
the
 
attention
 
on
 
semantic
 
professional
 
content.
 
GitHub  API  Rate  Limit  Strategies  
Unauthenticated  calls  to  the  GitHub  API  are  limited  to  60  per  hour,  which  can  be  exhausted  
quickly

are
 
using

from
 
the
 
same
 
server
 
IP.
 

Solution:  The  application  must  implement  a  token-based  sync  flow.  By  using  the  user's  own

from
 
the
 
server
 
to
 
the
 
individual
 
user,
 
providing
 
each
 
candidate
 
with
 
5,000
 
requests
 
per
 
hour—far

than
 
is
 
needed
 
for
 
a
 
single
 
resume
 
session.
